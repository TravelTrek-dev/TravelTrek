TravelTrek Project Documentation

وثيقة HTML منظمة إلى 4 أجزاء لتوثيق مشروع Flutter TravelTrek.

Part 1: Project Overview

TravelTrek هو تطبيق سفر ذكي مبني باستخدام Flutter، يهدف إلى توليد خطط سياحية مخصصة، إدارة الميزانية، وعرض تفاصيل الرحلة للمستخدم.

النقاط الأساسية

  • اسم المشروع: travel_trek
  • إطار العمل: Flutter
  • إصدارات مهمة: Dart SDK ^3.11.4
  • التبعيات الرئيسية: flutter_bloc, dio, get_it, firebase_auth, shared_preferences

الوظائف المتوقعة

  • توليد خطط سفر عبر API ذكي.
  • عرض تفاصيل الخطة ضمن شاشة Itinerary.
  • دعم تسجيل الدخول والحساب باستخدام Firebase أو بريد إلكتروني.
  • إدارة ميزانية الرحلة ونفقات السفر.

Part 2: Architecture & Modules

المشروع منظم بشكل أساسي إلى وحدات Features وطبقات Core لدعم بنية نظيفة ومفصولة.

هيكل المجلدات الرئيسة

  • lib/core: خدمات عامة، مساعدة، إعدادات Service Locator، وإدارة الأخطاء.
  • lib/Features: يحتوي على وحدات التطبيق الرئيسية مثل auth, home, Budget, Plandetails, plans, onboarding, splash.
  • lib/main.dart: نقطة البداية لتشغيل التطبيق.

نمط المعمارية

  • Repository: واجهات ومطبقات للوصول إلى البيانات.
  • Cubit / Bloc: لإدارة الحالة على مستوى الشاشة أو الميزة.
  • Models: تفكيك JSON إلى كائنات Dart (PlanModel, UserEntity, etc.).
  • Service Locator: استخدام GetIt لحقن التبعيات.

أهم الطبقات

  • Auth تسجيل وتشغيل المستخدم ونماذج UserEntity.
  • Home توليد خطة السفر وعرض القائمة.
  • Plandetails عرض تفاصيل الخطة، حفظ الخطة، والشاشات المتصلة.
  • Budget إضافة ومراجعة المصاريف.

Part 3: Core Flow and Features

تدفق توليد الخطة

  1. المستخدم يدخل وصف رحلة في شاشة Home.
  2. يُستدعى Cubit خاص بالخطة (GeneratePlanCubitCubit).
  3. يتم إرسال الطلب إلى HomeRepoImple.generatePlan باستخدام BackendService.generatePlanUrl.
  4. عند النجاح، ينتقل المستخدم إلى شاشة Itinerary مع PlanModel.

حفظ خطة الرحلة

  • تم إنشاء PlanRepo وPlanRepoImple في Features/Plandetails.
  • تعتمد عملية الحفظ على BackendService.savePlanUrl.
  • تستخدم PlanDetailsCubit حالة التحميل والنجاح والفشل.

نقاط اتصال API

  • BackendService.generatePlanUrl لتوليد خطط السفر.
  • BackendService.getPromptsUrl لجلب تلميحات وأفكار الرحلة.
  • BackendService.savePlanUrl لحفظ الخطة المولدة.
  • BackendService.addExpenseUrl وgetExpenseHistoryUrl لإدارة المصاريف.

Part 4: Setup & Usage

تشغيل المشروع

flutter pub get
flutter run

المتطلبات الأساسية

  • Flutter SDK متوافق مع 3.11.4 أو أحدث.
  • إعداد Firebase إذا كنت تستخدم تسجيل الدخول بجوجل أو البريد الإلكتروني.
  • تنفيذ Prefs.init() قبل استخدام بيانات SharedPreferences.

ملفات مهمة

  • lib/main.dart - نقطة البداية للتطبيق.
  • lib/core/services/service_locator.dart - تسجيل التبعيات.
  • lib/Features/home/presentation/views/widgets/home_view_body.dart - واجهة توليد الخطة.
  • lib/Features/Plandetails/presentation/views/widgets/itinerary_view.dart - شاشة تفاصيل الخطة.
